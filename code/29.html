▼ Algorithm Theory & Explanation

Two-Pass Algorithm

This algorithm uses a two-pass approach:

Pass 1: Count the number of hash (#) characters in the string.

Pass 2: Fill the result - place all hash characters at the beginning, then add non-hash characters maintaining their relative order.

Time Complexity

O(n) where n is the length of the string - we traverse the string twice.

Space Complexity

O(n) for the result array/string.

Edge Cases

  • String with no hash characters
  • String with all hash characters
  • Empty string
  • Single character string
