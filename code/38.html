Remove Adjacent Duplicates

Example: Input: "abbaca" → Output: "ca"

Theory & Background

The Remove Adjacent Duplicates problem can be efficiently solved using a stack data structure. The key insight is that when we encounter a character that matches the top of the stack, we have found an adjacent duplicate pair that needs to be removed.

This approach works because the stack maintains the characters we've seen so far, and when we find a match with the top element, we know we have adjacent duplicates (the top of stack and current character).

Algorithm Explanation

Algorithm Steps:

1. Initialize an empty stack

2. For each character in the input string:

- If stack is empty or current char ≠ top of stack: push current char

- If current char = top of stack: pop from stack (remove duplicate pair)

3. Convert remaining stack contents to string

Time Complexity: O(n) where n is the length of the string

Space Complexity: O(n) for the stack storage

Optimization Tricks

• Use the result string itself as a stack by treating it as a mutable character array

• Instead of using an actual stack data structure, use a pointer to track the "top" of our result

• In languages like C++, can modify the input string in-place to save space

• StringBuilder/StringBuffer can be used for better string concatenation performance

Edge Cases & Boundary Logic

Edge Cases:

• Empty string → return empty string

• Single character → return the character

• All characters are duplicates → return empty string

• No adjacent duplicates → return original string

• Nested duplicates like "aabbbaaa" → require multiple passes

Boundary Conditions:

• Stack empty check before accessing top element

• Proper handling of string indices

Implementation Caveats

• Be careful with string mutability in different languages

• Stack overflow possible with very long strings in recursive implementations

• Consider using deque/list instead of stack for languages without built-in stack

• Memory management important for languages like C/C++

• Unicode handling may require special consideration for multi-byte characters
